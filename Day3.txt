1. 🧭 Overview
1.1 Product Name

Academic Risk Lab: Breach Prevention (Day 3)

1.2 Objective

Build an interactive, gamified training simulation to help recruitment screening agents accurately validate education, graduation, and eligibility criteria.

1.3 Key Outcome

By the end of the experience, agents should be able to:

Identify academic eligibility issues
Detect inconsistencies across resume and application data
Make correct decisions under uncertainty
Apply escalation logic appropriately
Provide audit-ready justifications
2. 🎯 Problem Statement

Current training methods (MCQs, static exercises) fail to:

Simulate real-world ambiguity
Build decision-making skills
Reinforce accountability
Train agents to handle incomplete or conflicting data

This leads to:

Incorrect approvals
Unnecessary escalations
Audit risks
3. 💡 Solution

A simulation-based individual gameplay system where agents:

Investigate candidate education profiles
Reveal hidden data using limited evidence
Tag risks
Make final decisions (Proceed / Reject / Escalate)
Receive immediate feedback with consequences
4. 🎮 Experience Design
4.1 Experience Type
50% Realistic (screening dashboard)
50% Gamified (risk meter, scoring, time pressure)
4.2 Narrative Style

Light system-driven prompts:

“New Candidate Alert”
“Investigation Required”
“Potential Risk Detected”
“Decision Locked”
“Audit Risk Updated”
5. 👤 Target Users
Recruitment Screening Agents
New hires in screening roles
QA / Audit trainees
6. 🧩 Core Gameplay Loop
Candidate profile appears
Agent reviews visible data
Agent reveals hidden data (limited tokens)
Agent tags risks
Agent selects decision:
Proceed
Reject
Escalate
Agent provides justification
System gives feedback
Score + Risk Meter update
Move to next scenario

8. 🧠 Game Design: Timeline Collapse
8.1 Objective

Validate graduation timelines and detect inconsistencies

8.2 Scenarios (5 Total)
Scenario	Description	Decision
1	Resume vs portal mismatch (2023 vs 2024)	Escalate
2	Future graduation (2025)	Escalate
3	Backlog cleared after graduation year	Escalate
4	Applied before graduation completed	Escalate
5	Clean timeline	Proceed
9. 📊 Data Model
type Decision = "proceed" | "reject" | "escalate";

type RiskTag =
  | "degree"
  | "stream"
  | "cgpa"
  | "graduation"
  | "university"
  | "mismatch";

type EvidenceItem = {
  id: string;
  label: string;
  content: string;
};

type Scenario = {
  id: string;
  title: string;
  alert: string;
  difficulty: "easy" | "medium" | "hard";
  visibleData: {
    resume: string[];
    portal: string[];
  };
  evidenceTokens: number;
  evidence: EvidenceItem[];
  correctRisks: RiskTag[];
  correctDecision: Decision;
  strongJustificationKeywords: string[];
  misleadingClue: string;
  ruleApplied: string;
};
